## What This Is

A museum-grade listing detail experience for dealer listings with rich data. When a listing has enough content (provenance, sayagaki, koshirae, setsumei, video, etc.), the standard `/listing/[id]` page upgrades to an immersive dark-themed "Showcase" layout — a scholarly exhibit for a single sword.

The centerpiece is an **AI-generated Curator's Note** — a 2-3 paragraph scholarly narrative that synthesizes setsumei, sayagaki, artist biography from Yuhinkai, provenance, and designation statistics into contextual commentary that doesn't exist anywhere else.

## Architecture Decisions (Already Made)

### URL Strategy: Option D — Hybrid Adaptive Page
- **The Showcase lives at `/listing/[id]`** — the existing listing detail page dynamically upgrades to Showcase layout when `isShowcaseEligible(listing)` returns true
- **No new routes** — enhance `ListingDetailClient.tsx` with conditional layout branch
- **QuickView remains unchanged** for quick browsing, but gains a "View Exhibit →" CTA instead of "View full details" for Showcase-eligible listings
- **Full SSR** — Curator's Note, provenance, documentation all server-rendered for SEO
- **Graceful degradation** — if listing loses eligibility, page renders standard layout. No broken links
- Full analysis of all 4 options in `docs/SHOWCASE_QUICKVIEW.md` under "URL Strategy — Deep Analysis"

### Design Language: "Same Book, Different Subject"
- **Sibling of the artist page** (`/artists/[slug]`), not a departure — same typographic DNA
- Same gold rule dividers, small-caps section headers, metadata grids, serif display fonts, `space-y-16` rhythm, `max-w-[780px]` prose measure
- **Always-dark** atmosphere (forced dark regardless of system preference) — museum gallery at night
- Parchment-colored text cards (#f5f0e8) for documents (setsumei, sayagaki) create warm islands on dark background
- See "Design Tokens" section in main doc for exact CSS values

### Section Flow (9 sections)
1. **Hero** — Full-bleed image or muted autoplay video loop. Attribution text below in serif. `object-fit: contain` (blade silhouette floats on darkness)
2. **Identity Card** — Museum placard: type, attribution, school/era/province, measurements (2-col grid), mei type, nakago type, cert badge with session. Centered card on dark bg
3. **Curator's Note** — AI-generated scholarly prose. `max-w-prose`, drop cap, serif font. Loading skeleton while generating. "Generated by NihontoWatch AI" attribution
4. **Documentation** — Setsumei + Sayagaki + Hakogaki. Side-by-side on desktop (document image left 60%, translation text on parchment card right 40%). Stacked on mobile. Translation toggle for JA/EN
5. **Provenance Timeline** — Horizontal (desktop) / vertical (mobile). Dots on rule line. Gold dots when provenance images exist. Kiwame entries integrated as events
6. **Koshirae** — Full image + component cards (grid desktop, horizontal scroll mobile). Each card links to maker's artist page. Shows separate cert + setsumei if koshirae has its own designation
7. **Video Gallery** — If multiple videos. Primary player + thumbnail strip. Skipped if only 1 video used as hero
8. **Image Gallery** — Masonry grid (3-col desktop, 2-col mobile). Click → lightbox with pinch-zoom
9. **Dealer & Price** — Dealer identity (logo, name, domain), price, "Inquire" CTA, social sharing, endpiece rule

**Plus**: Sticky price bar (desktop: top bar after scrolling past hero; mobile: bottom bar) and SectionJumpNav (desktop: sticky nav with anchors: Overview · Scholar's Note · Documentation · Provenance · Mountings · Gallery)

### Video Integration
- Video can replace hero image as muted autoplay loop (cinematic opening)
- Additional videos in dedicated gallery section
- Video counts as a rich section in eligibility scoring
- Uses existing VideoGalleryItem/VideoPlayer components (restyle for dark theme)
- HLS via Bunny.net Stream (already configured)

### Activation Criteria
```typescript
function isShowcaseEligible(listing: Listing): boolean {
  const imageCount = (listing.images?.length ?? 0) + (listing.stored_images?.length ?? 0);
  const videoCount = listing.video_count ?? 0;
  if (imageCount < 5 && videoCount === 0) return false;

  let richSections = 0;
  if (listing.provenance?.length) richSections++;
  if (listing.sayagaki?.length) richSections++;
  if (listing.hakogaki?.length) richSections++;
  if (listing.koshirae?.components?.length || listing.koshirae?.artisan_id) richSections++;
  if (listing.kiwame?.length) richSections++;
  if (listing.setsumei_text_en || listing.setsumei_text_ja) richSections++;
  if (videoCount > 0) richSections++;

  if (listing.showcase_override === true) return true;
  if (listing.showcase_override === false) return false;

  return richSections >= 2;
}
```

Listing 90396 scores 6 rich sections — passes by wide margin.

### Curator's Note AI Generation
- **Input**: Assembles structured context from 8 data sources (sword data, artisan data from Yuhinkai, setsumei, sayagaki, hakogaki, kiwame, provenance, koshirae)
- **Model**: Via OpenRouter (same pattern as translate API). Model TBD — quality matters, this is the centerpiece
- **Rules**: NBTHK scholarly voice, quote don't rephrase, never fabricate, no price commentary, bilingual EN/JA, 2-3 paragraphs
- **Caching**: Store in DB columns, invalidate (NULL) when source data changes
- **Full generation spec**: See "Curator's Note — AI Scholarly Synthesis" section in `docs/SHOWCASE_QUICKVIEW.md`

## Critical Rules (From CLAUDE.md)

- **Rule #10**: NEVER use `{ passive: false }` on touchmove listeners (has broken the build 4 times)
- **Rule #9**: NEVER use fire-and-forget promises in API routes (Vercel serverless freezes after response)
- **Rule #8**: NEVER modify tests to match broken code
- **Enrichment fields ≠ DB columns**: `artisan_display_name`, `artisan_name_kanji` are runtime-only — verify against schema
- **Dual-path fields**: Swords use `smith`/`school`, Tosogu use `tosogu_maker`/`tosogu_school` — use `getAttributionName()`/`getAttributionSchool()` from `attribution.ts`
